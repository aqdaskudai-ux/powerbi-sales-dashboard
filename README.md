# 📊 Power BI Sales Analysis Dashboard

## 📌 Project Overview

This project focuses on building an end-to-end **Sales Analysis Dashboard** using Power BI.
The goal was to transform raw retail data into meaningful insights that support **business decision-making**.

The project covers the complete analytics workflow including **data cleaning, transformation, modeling, and visualization**.

---

## 📂 Dataset Description

The dataset used is **Superstore Sales Data** consisting of three tables:

* **Orders:** Contains transaction-level sales data
* **Returns:** Indicates returned orders
* **People:** Contains regional manager information

Key fields include:

* Sales, Profit, Quantity
* Customer details
* Product categories
* Geographic information (Region, State, City)

---

## ⚙️ Tools & Technologies

* **Power BI Desktop**
* **Power Query** (Data Cleaning & Transformation)
* **DAX** (Measures & Calculations)

---

## 🔧 Data Cleaning & Transformation

Data was prepared using Power Query with the following steps:

* Converted first row into headers
* Removed null and inconsistent values
* Renamed columns for better readability
* Created new columns using date functions
* Merged datasets to build relationships
* Applied transformations for structured analysis

---

## 🧩 Data Modeling

* Established relationships between **Orders, Returns, and People tables**
* Ensured proper data types and hierarchy (Year → Month → Date)
* Built a clean model to support efficient analysis

---

## 📊 Dashboard Structure

The dashboard consists of **5 interactive pages**:

### 1. Sales Overview

* KPI Cards: Total Sales, Profit, Quantity, Orders
* Sales by Region & Category
* Sales by Category (Pie Chart)
* Sales & Profit Trend (Yearly)
* Sales by State & Product

---

### 2. Profit Analysis

* Top 10 Products by Profit
* Profit by Sub-Category (Waterfall Chart)
* Profit Trend (Monthly/Quarterly)
* Drill-down analysis (Category → Product)

---

### 3. Time Series Analysis

* Sales trend over time with drill-down
* Performance threshold analysis
* Sales comparison by Region
* Segment-wise performance

---

### 4. Region Wise Analysis

* Sales by City and Category
* Sales by State and Region
* Geographic sales distribution (Treemap)
* Top-performing cities

---

### 5. Customer Analysis

* Top 10 Customers by Sales & Profit
* Sales by Customer Segments
* Detailed customer-level table

---

## 🎛️ Interactive Features

* Slicers for:

  * Date
  * Category
  * Region
  * Segment
* Drill-down functionality for deeper insights
* Cross-filtering across visuals

---

## 📈 Key Insights

* Sales show **seasonal variation**, with certain months underperforming
* A few categories contribute **majority of revenue**
* Some products generate **high sales but low profit**
* Sales are concentrated in specific **regions and cities**
* Customer contribution follows **Pareto principle (80/20 rule)**

---

## 🚀 Business Recommendations

* Focus on **high-profit products**, not just high sales
* Optimize pricing/discount strategy for low-margin products
* Expand operations in **top-performing regions**
* Target high-value customers with retention strategies
* Monitor monthly trends to manage seasonal fluctuations
